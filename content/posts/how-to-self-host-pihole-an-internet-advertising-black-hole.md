<p>This article will cover setting up your own self-hosted PiHole instance. Allowing you to run an internal DNS and block internet advertisements from reaching your machine.</p><h2 id="what-is-pihole">What is PiHole?</h2><p><a href="https://pi-hole.net/">PiHole</a> offers Network-wide protection for internet advertisements. PiHole saves you from needing to install an Ad-blocker on each device that you own. PiHole even blocks in-app advertisements, so no longer, are your free-to-play games interrupted by those annoying adverts.</p><figure class="kg-card kg-bookmark-card"><a class="kg-bookmark-container" href="https://pi-hole.net/"><div class="kg-bookmark-content"><div class="kg-bookmark-title">Home</div><div class="kg-bookmark-description">1. This means that whenever a new device connects to your router, it can automatically have adverts blocked. PiHole allows you to put devices into conditional groups. </p><p>Let's say that your partner wants to be able to click on their favourite Buzzfeed quizzes, to find out <a href="https://www.buzzfeed.com/mathewguiver/which-type-of-bread-are-you">what type of bread they are</a>. PiHole allows you to assign either a MAC Address or IP Address to be excluded from the PiHole advertisement block list.</p><figure class="kg-card kg-bookmark-card"><a class="kg-bookmark-container" href="https://www.buzzfeed.com/mathewguiver/which-type-of-bread-are-you"><div class="kg-bookmark-content"><div class="kg-bookmark-title">Which Type Of Bread Are You?</div><div class="kg-bookmark-description">No knead to worry.</div><div class="kg-bookmark-metadata"><img class="kg-bookmark-icon" src="https://www.buzzfeed.com/static-assets/bf-bpage-ui/_next/static/img/favicon.5a0c77a8815cfcc67c710199054a55c6.ico"><span class="kg-bookmark-author">BuzzFeed</span><span class="kg-bookmark-publisher">Mathew Guiver</span></div></div><div class="kg-bookmark-thumbnail"><img src="https://img.buzzfeed.com/buzzfeed-static/static/2020-12/18/5/campaign_images/a33e60e64038/which-type-of-bread-are-you-2-3667-1608270190-7_dblbig.jpg"></div></a></figure><p>PiHole also offers whitelists and blacklists. Meaning that if a domain slips through the PiHole net, you have the option of forcing a domain to be blocked. </p><p><em>This is great for when I've had enough of eating takeaways and end up blocking all of the common takeaway URLs on my internal network.</em></p><p>On the other hand, if there's a website that you frequent, which is being blocked. You can whitelist that site for your viewing pleasures.</p><h2 id="how-does-pihole-work">How does PiHole work?</h2><p>To understand PiHole, you need to understand DNS. DNS, or Domain Name System, is a service that is provided by a third party. When you, the computer user, want to navigate to a website, you would generally use a domain name to find the server which the website is hosted on.</p><p>DNS is the magic that prevents you from needing to remember the IP address associated with the Website.</p><p>So when you make a website request, to <a href="https://theselfhostingblog.com/">https://bowlerdesign.tech</a> for example, the DNS server will look through a list of domain and IP address mappings to find the IP address best associated with the domain name you're looking for. If the DNS server finds a match, it redirects you to the server which is hosting the website. If the DNS server can't find a match, then you might see an error along the lines of <code>ERR_NAME_NOT_RESOLVED</code></p><p><em>So what does PiHole have to do with this?</em></p><p>PiHole acts as your primary DNS server for your internal network. So as you're making requests on your local network, PiHole is checking every URL submitted, against the PiHole blocklist.</p><p>This means that if you navigate to a website that has advertisements, PiHole will spot that the advertisement URL is in the blocklist, and then sending back the <code>ERR_NAME_NOT_RESOLVED</code> error to us.</p><p>PiHole finally sends all the requests that have not been blocked, upstream to a DNS server of your choice. All of this in a matter of milliseconds.</p><h2 id="how-do-i-set-up-pihole">How do I set up PiHole?</h2><p>PiHole, hence the name, was originally intended to be served from a <a href="https://www.amazon.co.uk/gp/search?ie=UTF8&amp;tag=bowlerdesign-21&amp;linkCode=ur2&amp;linkId=f0502cacce9b5a8a52f19aadca703b8f&amp;camp=1634&amp;creative=6738&amp;index=computers&amp;keywords=Raspberry Pi">Raspberry Pi</a> but can now be set up on any kind of Linux enabled device.</p><figure class="kg-card kg-image-card"><img src="https://ir-uk.amazon-adsystem.com/e/ir?t=bowlerdesign-21&amp;l=ur2&amp;o=2" class="kg-image" alt loading="lazy"></figure><h3 id="step-1-set-up-the-server-you-want-to-run-pihole-on">Step 1: Set up the server you want to run PiHole on</h3><p>You'll need to either purchase a <a href="https://www.amazon.co.uk/gp/search?ie=UTF8&amp;tag=bowlerdesign-21&amp;linkCode=ur2&amp;linkId=838d86d54be7e521a789421988ebe7d1&amp;camp=1634&amp;creative=6738&amp;index=computers&amp;keywords=Raspberry Pi">Raspberry Pi</a> or use an existing server at your disposal.
