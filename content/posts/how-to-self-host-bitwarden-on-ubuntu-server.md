<h2 id="introduction">Introduction</h2><p>This article will cover setting up your own self-hosted Bitwarden (Vaultwarden) instance with Docker and configuring ngnix to allow for public exposure for cross-device access to your vault.</p><h3 id="what-is-bitwarden">What is Bitwarden?</h3><p><a href="https://bitwarden.com/">Bitwarden </a>is a free and open-source password management service that stores sensitive information such as website credentials in an encrypted vault. The Bitwarden platform offers a<a href="https://bitwarden.com/download/"> variety of client applications </a>including a web interface, desktop applications, browser extensions, mobile apps, and a CLI.</p><p>I use Bitwarden as my main password vault. It stores my card details for automating the filling out of payment forms. Saves me from having to find or remember my card details. I also use Bitwarden for storing all of my passwords. </p><p>Having Bitwarden as a public endpoint means that I can connect to my password vault using the Bitwarden app on Android, specifying my self hosted instance.</p><h2 id="setting-up-the-bitwarden-server">Setting up the Bitwarden Server</h2><p>This section of the tutorial is to set up the main Bitwarden 'hub'. This will be a publicly exposed Bitwarden API that will live on your server. </p><h3 id="step-1-setting-up-your-linux-server">Step 1: Setting up your Linux server</h3><p>You'll need to either have an existing server instance or create one. I use a Proxmox instance running on a server in my loft. You could also use something like <a href="https://www.digitalocean.com/">Digital Ocean</a> to host your Bitwarden Server. Using the following link will give you $100 worth of credits for 60 days to play around with, just sign up using <a href="https://m.do.co/c/d2a3afe52625">this link</a>.</p><p>You could also use a cheap <a href="https://amzn.to/3cWTlno">Raspberry PI</a> to set up your own Linux server.</p><p>Once you have the server set up, or have logged in. You'll need to do some updates and run some prerequisite installs.</p><pre><code class="language-bash">sudo apt-get update
sudo apt-get upgrade</code></pre><p>Next, we need to install Docker. Docker is the layer which your containers run. </p><p>To install Docker on your instance, you need to run the following command.</p><p>The following script is a convenience script <a href="https://docs.docker.com/engine/install/debian/#install-using-the-convenience-script"><strong>provided by the Docker team</strong></a>. It's highly recommended to always check what you're going to execute, before executing it.</p><figure class="kg-card kg-code-card"><pre><code class="language-bash">curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh</code></pre><figcaption>Installing Docker using the convenience script</figcaption></figure><p>Once you have executed the Docker install script. You should see an output like the following.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="/images/2021/02/image-8.png" class="kg-image" alt="Docker convenience script install output" loading="lazy"><figcaption>Docker convenience script install output</figcaption></figure><p>As you can see in the output, the command was executed successfully. You may also notice that there is a console message specifying how to use Docker as a non-root user.</p><p>This means that whenever you are executing the Docker command, you'll no longer need to type in your sudo password.</p><p>If this sounds good to you, you can simply run the provided command, substituting <code>your-user</code> for your server user. In my case, my user is <code>ubuntu</code>. My command would look like this.</p><figure class="kg-card kg-code-card"><pre><code class="language-bash">sudo usermod -aG docker ubuntu</code></pre><figcaption>Adding your user to the Docker group</figcaption></figure><p>We also need to install Docker Compose. This can be done by running the following commands.</p><pre><code class="language-bash">sudo curl -L "https://github.com/docker/compose/releases/download/1.28.5/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose</code></pre><h3 id="step-2-provisioning-your-bitwarden-server">Step 2: Provisioning your Bitwarden Server</h3><p>Next, you'll need to create a new folder, this will house your Bitwarden Server, you can call it anything memorable. I'll just call mine <code>bitwarden</code></p><pre><code class="language-bash">cd ~
mkdir bitwarden
cd bitwarden</code></pre><p>Next, you'll need to create a <code>docker-compose.yml</code> file. This is an orchestration file which <code>docker-compose</code> will use to provision your Docker instance.</p><pre><code class="language-bash">touch docker-compose.yml</code></pre><p>Next, you'll need to edit your `docker-compose.yml` file and paste in the following content.</p><pre><code class="language-yaml"># docker-compose.yml
version: '3'

services:
  bitwarden:
    image: vaultwarden/server:latest
    restart: always
    ports:
      - 8000:80
    volumes:
      - ./bw-data:/data
    environment:
      WEBSOCKET_ENABLED: 'true' # Required to use websockets
      SIGNUPS_ALLOWED: 'true'   # set to false to disable signups</code></pre><p>I'm using <a href="https://github.com/dani-garcia/vaultwarden">Vaultwarden </a> which is an opensource project. It is not owned by Bitwarden. They're an unofficial bitwarden compatible server written in Rust.</p><p>Save your <code>docker-compose.yml</code> file and exit back to your <code>bitwarden</code> directory.</p><h3 id="step-3-running-your-bitwarden-server-locally">Step 3: Running your Bitwarden Server locally</h3><p>Now, you have everything provisioned for running your Bitwarden Server.</p><p>The next thing to do is run it.</p><pre><code class="language-bash">sudo docker-compose up -d</code></pre><p>This will start up your Bitwarden Server inside Docker, it may take some time to pull down the images.</p><p>You can eventually see your instance running by executing the following</p><pre><code class="language-bash">sudo docker ps</code></pre><p>This will list your running instance.</p><p>If all is well, you can locally view your Bitwarden Server by navigating to <code>http://localhost:PORT</code>. Or from another machine by using your ip address instead of <code>localhost</code></p><p>You should see something that looks like the following.</p><figure class="kg-card kg-image-card"><img src="/images/2021/02/1.jpg" class="kg-image" alt loading="lazy" width="840" height="595" srcset="/images/size/w600/2021/02/1.jpg 600w, /images/2021/02/1.jpg 840w" sizes="(min-width: 720px) 720px"></figure><p>Finally, you'll just need to register for an account on your new hosted instance.</p><p>Click the <code>Create Account</code> button</p><p>Then fill out your details. If you have an existing Bitwarden account, you'll still have to create a new account on this instance. You can then Export and Import between accounts.</p><p>The last thing to do is hit <code>Submit</code></p><figure class="kg-card kg-image-card"><img src="/images/2021/02/2.jpg" class="kg-image" alt loading="lazy" width="833" height="940" srcset="/images/size/w600/2021/02/2.jpg 600w, /images/2021/02/2.jpg 833w" sizes="(min-width: 720px) 720px"></figure><p>If your instance isn't on your local machine, you will need to set up Nginx routing, which you can follow in Step 4.</p><h3 id="step-4-exposing-your-new-server-publicly">Step 4: Exposing your new server publicly</h3><p>This part may sound scary, but it is required to allow your Bitwarden Clients (Android, iOS, Chrome extension etc) to connect to your server.</p><p>We're going to be using nginx.</p><h4 id="setting-up-nginx">Setting up nginx</h4><p><a href="https://www.nginx.com/">Nginx</a> is a reverse proxy that allows you to point incoming web traffic to your new Bitwardeb server. </p><p>Firstly, install nginx if you haven't already</p><pre><code class="language-bash">sudo apt-get install nginx</code></pre><p>If you have UFW installed, you will have to Allow Nginx through your local firewall.</p><p>I have a tutorial for <a href="https://theselfhostingblog.com/posts/setting-up-ufw-on-ubuntu-server/">setting up UFW here</a></p><figure class="kg-card kg-bookmark-card"><a class="kg-bookmark-container" href="https://theselfhostingblog.com/posts/setting-up-ufw-on-ubuntu-server/"><div class="kg-bookmark-content"><div class="kg-bookmark-title">Setting Up UFW on Ubuntu Server</div><div class="kg-bookmark-description">UFW is a program that allows you to internally control ports on your Linux instance. This gives you the ability to forward ports from your machine.</div><div class="kg-bookmark-metadata"><img class="kg-bookmark-icon" src="https://bowlerdesign.tech/favicon.png"><span class="kg-bookmark-author">bowlerdesign.tech</span><span class="kg-bookmark-publisher">Ed Leeman</span></div></div><div class="kg-bookmark-thumbnail"><img src="https://images.unsplash.com/photo-1558494949-ef010cbdcc31?ixlib&#x3D;rb-1.2.1&amp;q&#x3D;80&amp;fm&#x3D;jpg&amp;crop&#x3D;entropy&amp;cs&#x3D;tinysrgb&amp;w&#x3D;2000&amp;fit&#x3D;max&amp;ixid&#x3D;eyJhcHBfaWQiOjExNzczfQ"></div></a></figure><pre><code class="language-bash">sudo ufw app list</code></pre><pre><code>Output
---

Available applications:
  Nginx Full
  Nginx HTTP
  Nginx HTTPS
  OpenSSH</code></pre><p>As you can see, there are three profiles available for Nginx:</p><ul><li><strong>Nginx Full</strong>: This profile opens both port 80 (normal, unencrypted web traffic) and port 443 (TLS/SSL encrypted traffic)</li><li><strong>Nginx HTTP</strong>: This profile opens only port 80 (normal, unencrypted web traffic)</li><li><strong>Nginx HTTPS</strong>: This profile opens only port 443 (TLS/SSL encrypted traffic)</li></ul><p>You can enable this by typing:</p><pre><code class="language-bash">sudo ufw allow 'Nginx Full'</code></pre><p>Next thing to do is just double check your nginx server is up and running</p><pre><code class="language-bash">sudo systemctl status nginx</code></pre><p>You should see something that looks like the following</p><pre><code>● nginx.service - A high performance web server and a reverse proxy server
   Loaded: loaded (/lib/systemd/system/nginx.service; enabled; vendor preset: enabled)
   Active: active (running)
...</code></pre><p>The next part allows us to take incoming traffic and point it to your container instance. Allowing you to expose your Bitwarden  server to the internet.</p><p>Navigate to <code>/etc/nginx/</code></p><pre><code class="language-shell">cd /etc/nginx/sites-enabled</code></pre><p>Use your favorite text editor and open the following file with sudo</p><pre><code class="language-shell">sudo vim default
</code></pre><p>I use the following code for my bitwarden server</p><pre><code class="language-shell">server {
    listen 80;
    server_name vault.bowlerdesign.tech;

    location / {
        proxy_pass http://127.0.0.1:8000; # bitwarden server address
    }
}
</code></pre><h4 id="port-forwarding">Port-forwarding</h4><p>You will need to port forward your instance to allow public access to your instance. This will involve googling how to port forward from your router.</p><p>You'll need to <a href="https://portforward.com/">point port 80 and 443 to your instance</a> where Nginx is set up.</p><h4 id="linking-bitwarden-server-with-your-public-domain">Linking Bitwarden Server with your public domain</h4><p>You will also need to set up a public domain name. This can then be used to call your new public instance with port 443 exposed.</p><p>For example, I would set up a subdomain on <code>bowlerdesign.tech</code> to be <code>vault.bowlerdesign.tech</code>. Notice this is also the domain I specified in my Nginx config above.</p><p>Here's something to search for with regards to <a href="https://www.google.com/search?client=firefox-b-d&amp;q=point+domain+name+to+server">setting up a domain name</a></p><h4 id="setting-up-certbot">Setting up Certbot</h4><p>Certbot allows us to generate SSL certificates for free with Let's Encrypt. It's simple to install and use. Even hooks in with Nginx, meaning that there's no more manual configuration required.</p><p>To install Certbot, simply run the following command</p><pre><code class="language-bash">sudo add-apt-repository ppa:certbot/certbot
sudo apt-get update
sudo apt-get install python-certbot-nginx</code></pre><p>Then, to set up your SSL certificate, run</p><pre><code class="language-bash">sudo certbot</code></pre><p>Follow the instructions, select your domain name from the nginx list.<br>Also, select <code>redirect</code> as this will upgrade any http requests to https.</p><h3 id="step-5-connecting-to-your-new-bitwarden-instance-from-a-client-">Step 5: Connecting to your new Bitwarden instance from a client.</h3><p>I'm going to use the Firefox Bitwarden Plugin for this part of the tutorial. But the process is identical for all Bitwarden clients.</p><p>First, if you haven't already, install your chosen Bitwarden client and open it.</p><p>In the top left corner, click the cog icon</p><figure class="kg-card kg-image-card"><img src="/images/2021/02/3.jpg" class="kg-image" alt loading="lazy" width="566" height="776"></figure><p>You'll then get some configuration. Simply add your full url into the <code>Server URL</code> field</p><figure class="kg-card kg-image-card"><img src="/images/2021/02/4.jpg" class="kg-image" alt loading="lazy" width="563" height="777"></figure><p>Like so, then just hit <code>Save</code> and log in as normal</p><figure class="kg-card kg-image-card"><img src="/images/2021/02/5.jpg" class="kg-image" alt loading="lazy" width="561" height="774"></figure><h3 id="that-s-it">That's it</h3><p>Pretty easy right? </p><p>Please don't hesitate to get in touch in the comments if you get stuck.
