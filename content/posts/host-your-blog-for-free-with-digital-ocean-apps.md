<figure class="kg-card kg-image-card kg-card-hascaption"><a href="https://bowlerdesign.co.uk?referrer=theselfhostingblog.com"><img src="/images/2021/09/Banner.png" class="kg-image" alt loading="lazy" width="500" height="350"></a><figcaption>Sponsored Post</figcaption></figure><h2 id="digital-ocean">Digital Ocean</h2><h3 id="what-is-digital-ocean-s-app-platform">What is Digital Ocean's App Platform?</h3><p><a href="https://www.digitalocean.com/">Digital Ocean</a> has introduced a new product called <a href="https://www.digitalocean.com/products/app-platform/">App Platform</a> which allows developers to quickly and easily build, deploy and scale apps. Digital Ocean will manage the solution for you, which means that the infrastructure, HTTPS, CDN and DNS routing is all done for you. This is awesome as it makes deploying apps simple.</p><p>With <a href="https://www.digitalocean.com/products/app-platform/">Digital Ocean's App Platform</a>, there is also the offer to host 3 static sites for free using their Starter pricing structure. </p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="/images/2021/03/image-12.png" class="kg-image" alt loading="lazy" width="925" height="724" srcset="/images/size/w600/2021/03/image-12.png 600w, /images/2021/03/image-12.png 925w" sizes="(min-width: 720px) 720px"><figcaption>Digital Ocean's Starter offerings</figcaption></figure><h3 id="what-is-a-static-site">What is a static site?</h3><p>A static website is essentially images and text. Each page is fixed with the content that was written by the author. Static sites are the most basic type of website and are the easiest to create.</p><h3 id="supported-static-site-generators">Supported static site generators</h3><p>Below are some examples of static site generators that <a href="https://www.digitalocean.com/docs/app-platform/languages-frameworks/">Digital Ocean supports</a></p><ul><li><a href="https://gohugo.io/">Hugo</a></li><li><a href="https://jekyllrb.com/">Jekyll</a></li><li><a href="https://nuxtjs.org/">Nuxt.js</a></li><li><a href="https://www.gatsbyjs.com/">Gatsby</a></li><li><a href="https://nextjs.org/">Next.js</a></li></ul><p>For this tutorial, we're going to be using Hugo!</p><h2 id="setting-up-your-static-hugo-site">Setting up your static Hugo site</h2><p>This is just going to be a brief walkthrough of how to get a Hugo site set up. I won't show you how to use Hugo. You can follow <a href="https://gohugo.io/getting-started/quick-start/">their quickstart guide</a> to get familiar with Hugo</p><h3 id="installing-hugo-locally">Installing Hugo locally</h3><p>To get up and running, we need to install the Hugo CLI on to our local environment. This will allow us to provision a new website, write posts and deploy it to our server. Don't panic though, this isn't where your website is going to be hosted. It's just where the tool lives to generate your website and write content</p><p>I'll be setting it up on my local Windows 10 machine with WSL installed. There are <a href="https://gohugo.io/getting-started/installing/">other setup instructions</a> if you're not using WSL Windows. Such as Homebrew.</p><p>Anyway, WSL instructions.</p><p>Make sure you're fully up to date.</p><pre><code class="language-bash">sudo apt-get update
sudo apt-get upgrade</code></pre><p>Next, we just need to pull down Hugo.</p><pre><code class="language-bash">sudo apt-get install hugo</code></pre><p>Once installed, we can create a new site. It doesn't matter where we create it, as we'll be committing it to Github to allow Digital Ocean to deploy our site!</p><pre><code class="language-bash">hugo new site example.com</code></pre><p>This will create a new directory, this is your website directory. </p><p>The Hugo output will look something like this.</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-13.png" class="kg-image" alt loading="lazy" width="749" height="280" srcset="/images/size/w600/2021/03/image-13.png 600w, /images/2021/03/image-13.png 749w" sizes="(min-width: 720px) 720px"></figure><p>Awesome! all good.</p><p>Now, we need to create a Git repository to allow us to commit our website to <a href="https://github.com/">Github</a>.</p><p>It's worth noting that Digital Ocean also supports <a href="https://about.gitlab.com/">GitLab</a>, so whichever you feel most comfortable with.</p><p>Change directory into your new website directory.</p><pre><code class="language-bash">cd example.com</code></pre><h2 id="configuring-source-control">Configuring source-control</h2><h3 id="initialising-git">Initialising Git</h3><p>Now we just need to initialise the Git repository.</p><pre><code class="language-bash">git init</code></pre><p>If you don't have git installed, just run the following.</p><pre><code class="language-bash">sudo apt-get install git</code></pre><h3 id="creating-a-new-repository">Creating a new repository</h3><p>We now need to navigate to GitHub (or GitLab) and create a new repository. This can be private if you want it to be!</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-14.png" class="kg-image" alt loading="lazy" width="756" height="771" srcset="/images/size/w600/2021/03/image-14.png 600w, /images/2021/03/image-14.png 756w" sizes="(min-width: 720px) 720px"></figure><p>Once configured, Create the repository!</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-15.png" class="kg-image" alt loading="lazy" width="1146" height="835" srcset="/images/size/w600/2021/03/image-15.png 600w, /images/size/w1000/2021/03/image-15.png 1000w, /images/2021/03/image-15.png 1146w" sizes="(min-width: 720px) 720px"></figure><p>Now, we need to follow the steps under the <code>...or push an existing repository from the command line</code></p><p>Back on WSL, we need to run the suggested commands to tie our local Git repository with the Github repository.</p><p>Make sure you're in the root of your website directory, where we added the <code>.git</code> file.</p><pre><code class="language-bash">cd example.com</code></pre><p>We now need to hook our Git repository up to Github.</p><p>Configure the origin.</p><pre><code class="language-bash">git remote add origin git@github.com:edleeman17/example.com.git</code></pre><p>Now, we need to get Hugo to generate the site for us.</p><pre><code class="language-bash">hugo -D</code></pre><p>This will produce a <code>public/</code> directory which will contain our static site files.</p><p>We now need to add all these files to our Git branch</p><pre><code class="language-shell">git add *</code></pre><p>Add a commit message</p><pre><code class="language-bash">git commit -m "Initial commit"</code></pre><p>Push our changes up to the origin.</p><pre><code class="language-bash">git push -u origin master</code></pre><p>Once pushed, you should see something like this on GitHub!</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-16.png" class="kg-image" alt loading="lazy" width="1199" height="545" srcset="/images/size/w600/2021/03/image-16.png 600w, /images/size/w1000/2021/03/image-16.png 1000w, /images/2021/03/image-16.png 1199w" sizes="(min-width: 720px) 720px"></figure><h2 id="setting-up-digital-ocean">Setting up Digital Ocean</h2><h3 id="creating-an-account">Creating an account</h3><p>You need to navigate to Digital Ocean and create a new account. Using <a href="https://m.do.co/c/d2a3afe52625">this link</a> will give you $100 worth of credits for 60 days to play around with. This is perfect if you want to use Digital Ocean for spinning up <a href="https://theselfhostingblog.com/posts/setting-up-a-valheim-server-using-docker/">Valheim servers</a> or <a href="https://theselfhostingblog.com/posts/how-to-self-host-bitwarden-on-ubuntu-server/">Bitwarden Instances</a>.</p><p>You don't need any credits to host your Hugo site.</p><h3 id="setting-up-a-new-app">Setting up a new App</h3><p>Once you have an account, you should <a href="https://cloud.digitalocean.com/apps">navigate to the Apps section</a>. And then click on Create App.</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-17.png" class="kg-image" alt loading="lazy" width="976" height="611" srcset="/images/size/w600/2021/03/image-17.png 600w, /images/2021/03/image-17.png 976w" sizes="(min-width: 720px) 720px"></figure><p>So, depending on which source you used, you need to click one of the sources shown.</p><p>We used GitHub, so I'm going to click GitHub.</p><p>Next, we need to select our repository.</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-18.png" class="kg-image" alt loading="lazy" width="773" height="676" srcset="/images/size/w600/2021/03/image-18.png 600w, /images/2021/03/image-18.png 773w" sizes="(min-width: 720px) 720px"></figure><p>And then the branch.</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-19.png" class="kg-image" alt loading="lazy" width="724" height="330" srcset="/images/size/w600/2021/03/image-19.png 600w, /images/2021/03/image-19.png 724w" sizes="(min-width: 720px) 720px"></figure><p>Digital Ocean will then detect that Hugo is the static site in your Git repository.</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-20.png" class="kg-image" alt loading="lazy" width="680" height="796" srcset="/images/size/w600/2021/03/image-20.png 600w, /images/2021/03/image-20.png 680w"></figure><p>You can leave all these values as default. Hit Next.</p><p>Now, give your site a name. This is just for internal purposes to Digital Ocean.</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-21.png" class="kg-image" alt loading="lazy" width="526" height="361"></figure><p>Finally, select the Starter package and hit Launch Starter App</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-22.png" class="kg-image" alt loading="lazy" width="835" height="724" srcset="/images/size/w600/2021/03/image-22.png 600w, /images/2021/03/image-22.png 835w" sizes="(min-width: 720px) 720px"></figure><h3 id="deploying-your-app">Deploying your App</h3><p>Once you have clicked Launch Starter App, you'll be taken to your App dashboard.</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-23.png" class="kg-image" alt loading="lazy" width="1158" height="679" srcset="/images/size/w600/2021/03/image-23.png 600w, /images/size/w1000/2021/03/image-23.png 1000w, /images/2021/03/image-23.png 1158w" sizes="(min-width: 720px) 720px"></figure><p>You can see that Digital Ocean is now building our app and deploying it. You'll know when your site is live when you see "Deployed successfully!" at the top of your dashboard.</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-24.png" class="kg-image" alt loading="lazy" width="1157" height="724" srcset="/images/size/w600/2021/03/image-24.png 600w, /images/size/w1000/2021/03/image-24.png 1000w, /images/2021/03/image-24.png 1157w" sizes="(min-width: 720px) 720px"></figure><p>Let's go see our website. Click on the Live App button.</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-25.png" class="kg-image" alt loading="lazy" width="641" height="420" srcset="/images/size/w600/2021/03/image-25.png 600w, /images/2021/03/image-25.png 641w"></figure><p>Sweet! we have a live URL! and an SSL certificate! Now, there's no content because we haven't configured any yet. Refer back to the Hugo quickstart guide for adding pages etc. </p><p>Make sure to commit your changes to the Git repository and Digital Ocean will automatically poll your Git repository and deploy any new changes.</p><h3 id="configuring-your-domain-name">Configuring your domain name</h3><p>We can now point a nicer looking domain name at our website. You should already have a domain name in mind. But if not, you can pick up a cheap one from <a href="https://www.namecheap.com/">NameCheap</a>.</p><p>On your App dashboard, click on the settings tab.</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-26.png" class="kg-image" alt loading="lazy" width="1187" height="696" srcset="/images/size/w600/2021/03/image-26.png 600w, /images/size/w1000/2021/03/image-26.png 1000w, /images/2021/03/image-26.png 1187w" sizes="(min-width: 720px) 720px"></figure><p>Down the list, you will find the Domains section. Click Edit.</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-27.png" class="kg-image" alt loading="lazy" width="1180" height="254" srcset="/images/size/w600/2021/03/image-27.png 600w, /images/size/w1000/2021/03/image-27.png 1000w, /images/2021/03/image-27.png 1180w" sizes="(min-width: 720px) 720px"></figure><p>Then click Add Domain.</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-28.png" class="kg-image" alt loading="lazy" width="927" height="832" srcset="/images/size/w600/2021/03/image-28.png 600w, /images/2021/03/image-28.png 927w" sizes="(min-width: 720px) 720px"></figure><p>Enter the domain name that you have just purchased.</p><p>You have two options here, either allow Digital Ocean to manage your DNS records and point your domain to their server. Or manually manage your domain to point to their servers.</p><p>Out of simplicity, I'm going to allow Digital Ocean to manage my domain.</p><p>You need to log into your domain name provider and change the nameservers to point to Digital Ocean.</p><ul><li>ns1.digitalocean.com</li><li>ns2.digitalocean.com</li><li>ns3.digitalocean.com</li></ul><p>Once done, Add your domain.</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-29.png" class="kg-image" alt loading="lazy" width="1160" height="309" srcset="/images/size/w600/2021/03/image-29.png 600w, /images/size/w1000/2021/03/image-29.png 1000w, /images/2021/03/image-29.png 1160w" sizes="(min-width: 720px) 720px"></figure><p>You'll see that the custom domain name is now pending. Now, DNS is a magical thing, it'll take 24 hours for your domain name to point to your Digital Ocean app. Just give it time, try navigating from a different machine, incognito, from 4G etc.. eventually, you will see your site pop up.</p><h2 id="the-long-wait">The long wait</h2><p>Wait for your DNS to propagate, after that, there's not much more to it. Simply use the Hugo tool to write your posts. Commit to GitHub. Enjoy your site.</p><p>Thanks for reading along, hopefully, it helped you out. Feel free to add any comments down below!</p>
