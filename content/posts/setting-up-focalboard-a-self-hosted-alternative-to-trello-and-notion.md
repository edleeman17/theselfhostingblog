<h2 id="focalboard">Focalboard</h2><p><a href="https://www.focalboard.com">Focalboard</a> is a self-hosted <a href="https://github.com/mattermost/focalboard">open-source</a> alternative to tools such as <a href="https://www.notion.so">Notion</a> or <a href="https://trello.com">Trello</a> which you can run on your own instance. Focalboard helps with tracking and organising your own work, as well as across a team.</p><p>Focalboard is currently in early-access beta, so this post may change over time. We'll try out best to update our tutorial as the project becomes more mature.</p><p>You can read more on Focalboard on their site <a href="https://www.focalboard.com">https://www.focalboard.com</a>.</p><p>In this tutorial, we're going to be setting up the Focalboard Personal Server, which can be installed on a Ubuntu server to be used within a team either locally, or exposed as a public endpoint using Nginx.</p><h2 id="installing-focalboard">Installing Focalboard</h2><h3 id="prerequisites-provisioning-your-server">Prerequisites: Provisioning your server</h3><p>You'll need to either purchase something like a <a href="https://www.amazon.co.uk/gp/search?ie=UTF8&amp;tag=bowlerdesign-21&amp;linkCode=ur2&amp;linkId=838d86d54be7e521a789421988ebe7d1&amp;camp=1634&amp;creative=6738&amp;index=computers&amp;keywords=Raspberry%20Pi"><strong><strong>Raspberry Pi</strong></strong></a> with Ubuntu Server installed or use an existing server at your disposal.</p><p>I use a <a href="https://www.proxmox.com/en/">Proxmox</a> instance running on a server in my loft.</p><p>You could also use something like <a href="https://www.digitalocean.com/"><strong><strong>Digital Ocean</strong></strong></a> to run your Docker server, Using the following link will give you $100 worth of credits for 60 days to play around with, just sign up using <a href="https://m.do.co/c/d2a3afe52625"><strong><strong>this link</strong></strong></a>.</p><p>If you are using Digital Ocean, be sure to follow our tutorials on <a href="https://theselfhostingblog.com/posts/setting-up-ufw-on-ubuntu-server/"><strong>locking down your exposed ports with UFW</strong></a> and <a href="https://theselfhostingblog.com/posts/self-hosting-a-wireguard-vpn-the-easy-way/"><strong>setting up a Wireguard VPN</strong></a> to ensure that your Digital Ocean endpoints are protected.</p><h3 id="step-1-updating-your-server-to-the-latest-version">Step 1: Updating your server to the latest version</h3><p>All of the following instructions are based on the Debian distro, so if you're running a server with Ubuntu, these instructions will be perfect for you. If not, you may have to adjust the commands below to suit your distro.</p><p>The first step is to just make sure our server is up to date. Run the following commands to pull down the latest updates from our distro repositories.</p><pre><code class="language-bash">sudo apt-get update &amp;&amp; sudo apt-get upgrade</code></pre><p>You should see an output like the following</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="/images/2021/02/image-7.png" class="kg-image" alt="Console output for running update and upgrade commands" loading="lazy"><figcaption>Console output for running the update and upgrade commands</figcaption></figure><h3 id="step-2-installing-the-focalboard-personal-server">Step 2: Installing the Focalboard Personal Server</h3><p>Firstly, we need to download the latest release package from the Focalboard GitHub release page. This can be done using the <code>wget</code> command.</p><p>At the time of writing this, the latest version is <code>v0.6.1</code> so this is the command that we will be using for this version. You may want to check the version of the latest release of Focalboard <a href="https://github.com/mattermost/focalboard/releases">here</a> and then update the following command accordingly.</p><pre><code class="language-bash">wget https://github.com/mattermost/focalboard/releases/download/v0.6.1/focalboard-server-linux-amd64.tar.gz</code></pre><p>After executing that command, run <code>ls</code> to make sure that your file is present.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="/images/2021/03/image-32.png" class="kg-image" alt loading="lazy" width="1120" height="94" srcset="/images/size/w600/2021/03/image-32.png 600w, /images/size/w1000/2021/03/image-32.png 1000w, /images/2021/03/image-32.png 1120w" sizes="(min-width: 720px) 720px"><figcaption>Ignore snap</figcaption></figure><p>Now, we need to extract this <code>tar.gz</code> file. You can do so by running the following command.</p><pre><code class="language-bash">tar -xvzf focalboard-server-linux-amd64.tar.gz</code></pre><p>You should now see this when running <code>ls</code> again</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-33.png" class="kg-image" alt loading="lazy" width="1127" height="336" srcset="/images/size/w600/2021/03/image-33.png 600w, /images/size/w1000/2021/03/image-33.png 1000w, /images/2021/03/image-33.png 1127w" sizes="(min-width: 720px) 720px"></figure><p>As you can see, we have now exported the Focalboard executable. We now need to move it to our <code>/opt</code> folder on our system.</p><pre><code class="language-bash">sudo mv focalboard /opt</code></pre><h3 id="step-3-installing-and-configuring-nginx">Step 3: Installing and configuring Nginx</h3><p>Nginx is a reverse proxy that allows us to point a domain name to our Focalboard service. By default, Focalboard will be running on port 8000. In theory, you could skip this step and access Focalboard using your server's IP address and port 8000.</p><p>We're going to use Nginx so that we can use a domain name instead.</p><p>First, we need to install Nginx.</p><pre><code class="language-bash">sudo apt-get install nginx</code></pre><p>Once installed, we need to configure a new site. This is essentially a config file that will allow routing on a certain domain name to our Focalboard service.</p><p>Usually, I name my sites by their URL. To create a new site, run the following command, substituting my URL with yours.</p><pre><code class="language-bash">sudo nano /etc/nginx/sites-available/focalboard.theselfhostingblog.com</code></pre><p>This will open up a new window that allows us to add our Nginx configuration. Now, you're about to paste in a lot of text, but don't worry, I'll walk you through what we need to change.</p><p>Paste in the following code.</p><pre><code class="language-bash">upstream focalboard {
   server localhost:8000;
   keepalive 32;
}

server {
   listen 80 default_server;

   server_name focalboard.theselfhostingblog.com; #Change Me

   location ~ /ws/* {
       proxy_set_header Upgrade $http_upgrade;
       proxy_set_header Connection "upgrade";
       client_max_body_size 50M;
       proxy_set_header Host $http_host;
       proxy_set_header X-Real-IP $remote_addr;
       proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
       proxy_set_header X-Forwarded-Proto $scheme;
       proxy_set_header X-Frame-Options SAMEORIGIN;
       proxy_buffers 256 16k;
       proxy_buffer_size 16k;
       client_body_timeout 60;
       send_timeout 300;
       lingering_timeout 5;
       proxy_connect_timeout 1d;
       proxy_send_timeout 1d;
       proxy_read_timeout 1d;
       proxy_pass http://focalboard;
   }

   location / {
       client_max_body_size 50M;
       proxy_set_header Connection "";
       proxy_set_header Host $http_host;
       proxy_set_header X-Real-IP $remote_addr;
       proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
       proxy_set_header X-Forwarded-Proto $scheme;
       proxy_set_header X-Frame-Options SAMEORIGIN;
       proxy_buffers 256 16k;
       proxy_buffer_size 16k;
       proxy_read_timeout 600s;
       proxy_cache_revalidate on;
       proxy_cache_min_uses 2;
       proxy_cache_use_stale timeout;
       proxy_cache_lock on;
       proxy_http_version 1.1;
       proxy_pass http://focalboard;
   }
}</code></pre><p>Most of this is configured for you and doesn't need to be changed. Only the URL which Nginx wants to use. You'll see that mine is <code>focalboard.theselfhostingblog.com</code> make sure that you change this to your URL. Also, don't forget the <code>;</code> at the end of the line. It'll save you a lot of headaches 😉</p><p>We now need to hit <code>Ctrl + x</code> ,<code>y</code> and <code>return</code> to save and exit out of <code>nano</code>.</p><p>Now, we just need to register our new site with Nginx, you can do this by running the following command. Again substituting my domain with yours.</p><pre><code class="language-bash">sudo ln -s /etc/nginx/sites-available/focalboard.theselfhostingblog.com /etc/nginx/sites-enabled/focalboard.theselfhostingblog.com</code></pre><p>We just need to remove the default Nginx configuration to prevent a port clash.</p><pre><code class="language-bash">sudo rm /etc/nginx/sites-available/default
sudo rm /etc/nginx/sites-enabled/default</code></pre><p>Now, we can test the Nginx configuration. Run the following command.</p><pre><code class="language-bash">sudo nginx -t</code></pre><p>Which should give us a successful output.</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-34.png" class="kg-image" alt loading="lazy" width="1137" height="90" srcset="/images/size/w600/2021/03/image-34.png 600w, /images/size/w1000/2021/03/image-34.png 1000w, /images/2021/03/image-34.png 1137w" sizes="(min-width: 720px) 720px"></figure><p>Sweet. Let's restart the Nginx service.</p><pre><code class="language-bash">sudo systemctl restart nginx</code></pre><h3 id="step-4-port-forwarding">Step 4: Port-forwarding</h3><p>If you're self-hosting this on your own hardware, you will need to port forward your instance to allow public access to your instance. This will involve googling how to port forward from your router.</p><p>You'll need to <a href="https://portforward.com/"><strong>point port 80 and 443 to your instance</strong></a> where Nginx is set up.</p><p>If you're hosting Focalboard in Digital Ocean, you may need to configure UFW.</p><p>I have a tutorial for <a href="https://theselfhostingblog.com/posts/setting-up-ufw-on-ubuntu-server/"><strong>setting up UFW here</strong></a></p><pre><code class="language-bash">sudo ufw app list</code></pre><pre><code class="language-bash">Output
---

Available applications:
  Nginx Full
  Nginx HTTP
  Nginx HTTPS
  OpenSSH</code></pre><p>As you can see, there are three profiles available for Nginx:</p><ul><li><strong>Nginx Full</strong>: This profile opens both port 80 (normal, unencrypted web traffic) and port 443 (TLS/SSL encrypted traffic)</li><li><strong>Nginx HTTP</strong>: This profile opens only port 80 (normal, unencrypted web traffic)</li><li><strong>Nginx HTTPS</strong>: This profile opens only port 443 (TLS/SSL encrypted traffic)</li></ul><p>It is recommended that you enable the most restrictive profile that will still allow the traffic you’ve configured. Since we will be configuring SSL for our server we will need to temporarily allow port 80 and 443 for Certbot to verify our domain endpoint.</p><p>You can enable this by typing:</p><pre><code class="language-bash">sudo ufw allow 'Nginx Full'</code></pre><h3 id="step-5-configuring-certbot">Step 5: Configuring Certbot</h3><p>Certbot allows us to generate SSL certificates for free with Let's Encrypt. It's simple to install and use. Even hooks in with Nginx, meaning that there's no more manual configuration required.</p><p>To install Certbot, simply run the following commands</p><pre><code class="language-bash">sudo apt-get install certbot python3-certbot-nginx</code></pre><p>Then, to set up your SSL certificate, run. Make sure that you have your domain name pointing to your IP address. This will require some DNS configuration. You'll need an A Record.</p><pre><code class="language-bash">sudo certbot --nginx</code></pre><p>Follow the instructions, select your domain name from the Nginx list.<br>Also, select <code>redirect</code> as this will upgrade any HTTP requests to HTTPS.</p><p>Once done, if you navigate to your domain, you should see the following page with a 502 Bad Gateway. This is fine as Focalboard isn't running at the moment. You should also notice that the SSL certificate is causing your domain to be HTTPS!</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-35.png" class="kg-image" alt loading="lazy" width="490" height="69"></figure><h2 id="installing-postgresql">Installing Postgresql</h2><p>For your server, it is recommended to use Postgres to store your data in a production environment. To install Postgres, you need to run the following command.</p><pre><code class="language-bash">sudo apt install postgresql postgresql-contrib</code></pre><p>Once installed, we need to log in as the Postgres user to configure our database.</p><pre><code class="language-bash">sudo --login --user postgres</code></pre><p>And now, we need to execute into the Postgres helper.</p><pre><code class="language-bash">psql</code></pre><p>You should now be in the Postgres prompt.</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-36.png" class="kg-image" alt loading="lazy" width="1127" height="100" srcset="/images/size/w600/2021/03/image-36.png 600w, /images/size/w1000/2021/03/image-36.png 1000w, /images/2021/03/image-36.png 1127w" sizes="(min-width: 720px) 720px"></figure><p>Let's create our database to store our Focalboard data.</p><pre><code class="language-bash">CREATE DATABASE boards;</code></pre><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-37.png" class="kg-image" alt loading="lazy" width="347" height="78"></figure><p>Now, we need to create our user.</p><pre><code class="language-bash">CREATE USER ed WITH PASSWORD 'secretpassword';</code></pre><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-38.png" class="kg-image" alt loading="lazy" width="607" height="83" srcset="/images/size/w600/2021/03/image-38.png 600w, /images/2021/03/image-38.png 607w"></figure><p>Give yourself a username and a secure password, please don't copy mine.</p><p>We can now exit the Postgres prompt by typing <code>exit</code>.</p><p>Then type <code>exit</code> again to exit the Postgres user context.</p><p>We now need to tell Focalboard to use the new user we just created. This can be done by updating the Focalboard config file. Type the following command to edit the <code>config.json</code> file.</p><pre><code class="language-bash">nano /opt/focalboard/config.json</code></pre><p>You'll open a file that looks like the following.</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-39.png" class="kg-image" alt loading="lazy" width="1119" height="398" srcset="/images/size/w600/2021/03/image-39.png 600w, /images/size/w1000/2021/03/image-39.png 1000w, /images/2021/03/image-39.png 1119w" sizes="(min-width: 720px) 720px"></figure><p>We need to replace lines 4 and 5 with the following to hook into our Postgres database.</p><pre><code>"dbtype": "postgres",
"dbconfig": "postgres://ed:secretpassword@localhost/boards?sslmode=disable&amp;connect_timeout=10",</code></pre><p>Make sure to update the <code>dbconfig</code> with your username and password.</p><p>It'll look something like the following.</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-40.png" class="kg-image" alt loading="lazy" width="1130" height="485" srcset="/images/size/w600/2021/03/image-40.png 600w, /images/size/w1000/2021/03/image-40.png 1000w, /images/2021/03/image-40.png 1130w" sizes="(min-width: 720px) 720px"></figure><p>Save and exit <code>nano</code> like we did before.</p><h2 id="running-focalboard-as-a-service">Running Focalboard as a service</h2><p>Nearly there. We now need to configure Focalboard to run as a service on our server. This means that Focalboard will automatically start when our server boots up. To do so, run the following command.</p><pre><code class="language-bash">sudo nano /lib/systemd/system/focalboard.service</code></pre><p>Once again, this will open up a <code>nano</code> window, we just need to paste the following in.</p><pre><code class="language-bash">[Unit]
Description=Focalboard server

[Service]
Type=simple
Restart=always
RestartSec=5s
ExecStart=/opt/focalboard/bin/focalboard-server
WorkingDirectory=/opt/focalboard

[Install]
WantedBy=multi-user.target</code></pre><p>This is some instructions on how to start up the Focalboard service.</p><p>Now, we can start the Focalboard service. Execute the following commands.</p><pre><code class="language-bash">sudo systemctl daemon-reload
sudo systemctl start focalboard.service
sudo systemctl enable focalboard.service</code></pre><h2 id="configuring-the-focalboard-ui">Configuring the Focalboard UI</h2><p>Now, if you kept your browser window open, you might see the page has loaded!</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-41.png" class="kg-image" alt loading="lazy" width="458" height="45"></figure><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-42.png" class="kg-image" alt loading="lazy" width="516" height="483"></figure><p>Let's create a new account.</p><p>Fill in your details and hit register.</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-43.png" class="kg-image" alt loading="lazy" width="491" height="450"></figure><p>Once registered, you'll be greeted with a blank page.</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-44.png" class="kg-image" alt loading="lazy" width="1251" height="737" srcset="/images/size/w600/2021/03/image-44.png 600w, /images/size/w1000/2021/03/image-44.png 1000w, /images/2021/03/image-44.png 1251w" sizes="(min-width: 720px) 720px"></figure><p>Don't panic, we just need to add a board.</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-45.png" class="kg-image" alt loading="lazy" width="316" height="421"></figure><p>Click on a template and load up your new Focalboard!</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-46.png" class="kg-image" alt loading="lazy" width="1747" height="748" srcset="/images/size/w600/2021/03/image-46.png 600w, /images/size/w1000/2021/03/image-46.png 1000w, /images/size/w1600/2021/03/image-46.png 1600w, /images/2021/03/image-46.png 1747w" sizes="(min-width: 720px) 720px"></figure><p>The first user registration will always be permitted, but <strong>subsequent registrations will require an invite link that includes a code</strong>. You can invite additional users by clicking on your username in the top left, then selecting “Invite users”.</p><h2 id="thats-all">That's all</h2><p>You can refer to Focalboard's <a href="https://www.focalboard.com/guide/user/">User's Guide</a> to find out more about how to use Focalboard.</p><p>To recap, we have...</p><ul><li>Spun up a new server running Ubuntu</li><li>Installed Focalboard onto our server</li><li>Configured Postgres as our database</li><li>Configured Focalboard as a service</li><li>Install an SSL Certificate for HTTPS</li><li>Configured a reverse proxy to point our domain to our running service</li><li>Created a new account on Focalboard</li></ul><p>That's pretty good going!</p><p>Let me know in the comments if you get stuck along the way, I'll do everything I can to help you out.
