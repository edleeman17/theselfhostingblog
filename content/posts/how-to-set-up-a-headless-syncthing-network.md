<figure class="kg-card kg-image-card kg-card-hascaption"><a href="https://bowlerdesign.co.uk?referrer=theselfhostingblog.com"><img src="/images/2021/09/Banner.png" class="kg-image" alt loading="lazy" width="500" height="350"></a><figcaption>Sponsored Post</figcaption></figure><h2 id="overview-of-syncthing">Overview of Syncthing</h2><p><a href="https://syncthing.net/">Syncthing</a> is a P2P (peer to peer) network that allows you to keep your files synchronised.</p><p>Syncthing aims to replace services such as: Dropbox, Google Drive and OneDrive. By putting the user in control of their own data. Syncthing enables the user to set up multiple 'nodes' which can communicate with eachother.</p><p>A node can be, your personal computer, a work machine, a machine in another location or even a central server.</p><p>Syncthing will encrypt all data when it is being transported to your other nodes. Nodes don't just have to be on your local network, you can sync between nodes over the internet. Encrypted, of-course.</p><h2 id="use-case">Use case</h2><p>I use Syncthing as my storage solution. I have a 'central' Syncthing server which lives in my loft. I quoted 'central' as the loft server is just a node, there's nothing special about it other than having a backup cron job running on it.</p><p>I then have multiple other nodes, such as: my main laptop, a work machine and even my Android phone. I have spoken about Syncthing on Android in my <a href="https://theselfhostingblog.com/posts/my-self-hosted-note-syncing-journey-once-switching-to-iphone/">My Self Hosted Note Syncing Journey Once Switching to Iphone</a> post.</p><p>With the multiple nodes set up, I can choose folders that can sync. I'm currently writing this blog post on my personal laptop, which is syncing to my blog server as I type (well, hit save).</p><p>You can then specify which nodes, the folder you created, to be shared with. So in my case, I am sharing my blog with my personal laptop and my 'central' server, just to backup my data.</p><hr><h2 id="setting-up-syncthing">Setting up Syncthing</h2><p>I'm going to show you how to set up Syncthing in the context of my setup. So having that 'central' server. You could use a cheap <a href="https://amzn.to/3cWTlno">Raspberry PI</a> to set up your 'central' server.</p><p>My 'central' server is running Ubuntu Server, so the following instructions are tailored for Debian distros.</p><p>Firstly, you need to install the correct package locations in order to download Syncthing correctly.</p><pre><code>sudo apt-get install curl apt-transport-https

curl -s https://syncthing.net/release-key.txt | sudo apt-key add -

echo "deb https://apt.syncthing.net/ syncthing stable" | sudo tee /etc/apt/sources.list.d/syncthing.list
</code></pre><p>Next, update your local repository with the new Syncthing repository</p><pre><code>sudo apt-get update
sudo apt-get upgrade

sudo apt-get install syncthing
</code></pre><p>The next thing you want to do is enable Syncthing as a system service. This will allow start-up on boot and continuous execution in the background of your server.</p><pre><code>sudo systemctl enable syncthing@username.service

sudo systemctl start syncthing@username.service
</code></pre><p>So this will start Syncthing on your Ubuntu Server. Problem is, Ubuntu Server is usually headless. So at the moment there is no way to access the Syncthing GUI to configure your folders and nodes.</p><p>The way we can change this to allow local access from another machine is to change the serve address.</p><p>When you ran the Syncthing command above, a <code>syncthing</code> folder was created under your <code>/home/username/.config</code> folder.</p><p>Inside that will be a <code>config.xml</code> file.</p><p>Open this with a text editor, something like Vim or Nano. It really doesn't matter.</p><p>Within <code>config.xml</code> there is a section that looks like this:</p><pre><code>...
&lt;gui enabled="true" tls="false" debugging="false"&gt;
    &lt;address&gt;127.0.0.1:8384&lt;/address&gt;
    &lt;apikey&gt;...&lt;/apikey&gt;
    &lt;theme&gt;default&lt;/theme&gt;
&lt;/gui&gt;
...
</code></pre><p>Under the address attribute, you need to update the ip address.<br><code>0.0.0.0</code> will allow all machines on your local network to be able to access the Syncthing GUI.</p><pre><code>...
&lt;gui enabled="true" tls="false" debugging="false"&gt;
    &lt;address&gt;0.0.0.0:8384&lt;/address&gt;
    &lt;apikey&gt;...&lt;/apikey&gt;
    &lt;theme&gt;default&lt;/theme&gt;
&lt;/gui&gt;
...
</code></pre><p>Save and close <code>config.xml</code> and then restart Syncthing by running the following command.</p><pre><code>sudo systemctl restart syncthing@username.service
</code></pre><h2 id="second-node">Second Node</h2><blockquote>For Syncthing to work, you need to set up another Syncthing node. Simply follow the above steps again on another machine, or jump to the bottom of this page quickly and install Syncthing on Windows or Android</blockquote><hr><h2 id="accessing-the-syncthing-gui">Accessing the Syncthing GUI</h2><p>Now with all that running, you should be able to access the Syncthing GUI by navigating to:</p><pre><code>http://{your_server_ip}:8384
</code></pre><p>Once entering your ip-address or hostname, you should be presented with something that looks like this</p><figure class="kg-card kg-image-card"><img src="/images/2021/02/1.jpeg" class="kg-image" alt="syncthing-gui" loading="lazy" width="2000" height="1009" srcset="/images/size/w600/2021/02/1.jpeg 600w, /images/size/w1000/2021/02/1.jpeg 1000w, /images/size/w1600/2021/02/1.jpeg 1600w, /images/2021/02/1.jpeg 2000w" sizes="(min-width: 720px) 720px"></figure><h3 id="adding-devices-nodes-">Adding Devices (nodes)</h3><figure class="kg-card kg-image-card"><img src="/images/2021/02/2.jpeg" class="kg-image" alt="syncthing-remote-device" loading="lazy" width="1275" height="543" srcset="/images/size/w600/2021/02/2.jpeg 600w, /images/size/w1000/2021/02/2.jpeg 1000w, /images/2021/02/2.jpeg 1275w" sizes="(min-width: 720px) 720px"></figure><p>To add a new Device, select the <code>Add Remote Device</code> button, which will open up a new popup window.</p><figure class="kg-card kg-image-card"><img src="/images/2021/02/3.jpeg" class="kg-image" alt="syncthing-add-device" loading="lazy" width="1819" height="982" srcset="/images/size/w600/2021/02/3.jpeg 600w, /images/size/w1000/2021/02/3.jpeg 1000w, /images/size/w1600/2021/02/3.jpeg 1600w, /images/2021/02/3.jpeg 1819w" sizes="(min-width: 720px) 720px"></figure><p>The <code>Add Device</code> window will pop up. You may notice that there is an existing ID which is available to click.<br>Syncthing automatically polls your local network looking for existing nodes, depending on how your other nodes are set up, you may or may not see this.</p><figure class="kg-card kg-image-card"><img src="/images/2021/02/4.jpeg" class="kg-image" alt="syncthing-device-id" loading="lazy" width="1361" height="861" srcset="/images/size/w600/2021/02/4.jpeg 600w, /images/size/w1000/2021/02/4.jpeg 1000w, /images/2021/02/4.jpeg 1361w" sizes="(min-width: 720px) 720px"></figure><p>If you don't see the ID, simply navigate to your other Syncthing node (let's call it <code>Syncthing B</code>), select <code>Actions &gt; Show ID</code> from the top right menu.</p><figure class="kg-card kg-image-card"><img src="/images/2021/02/5.jpeg" class="kg-image" alt="syncthing-device-id-qr" loading="lazy" width="1836" height="1208" srcset="/images/size/w600/2021/02/5.jpeg 600w, /images/size/w1000/2021/02/5.jpeg 1000w, /images/size/w1600/2021/02/5.jpeg 1600w, /images/2021/02/5.jpeg 1836w" sizes="(min-width: 720px) 720px"></figure><p>Here you will see the <code>Syncthing B</code> Syncthing ID. Copy this ID or use the QR code to scan the ID. And paste the ID into the <code>Device ID</code> field back on the <code>Syncthing A</code> node.</p><p>Complicated at first, but once you get the hang of Syncthing, it'll be easier.</p><p>Click <code>Save</code> on that window and it'll add <code>Syncthing B</code> as a new device.</p><h3 id="sharing-folders">Sharing Folders</h3><figure class="kg-card kg-image-card"><img src="/images/2021/02/6.jpeg" class="kg-image" alt="syncthing-new-folder" loading="lazy" width="1169" height="375" srcset="/images/size/w600/2021/02/6.jpeg 600w, /images/size/w1000/2021/02/6.jpeg 1000w, /images/2021/02/6.jpeg 1169w" sizes="(min-width: 720px) 720px"></figure><p>Now, on <code>Syncthing A</code> click the <code>Add Folder</code> button on the left hand side of the screen. This will pop up with a new window.</p><figure class="kg-card kg-image-card"><img src="/images/2021/02/7.jpeg" class="kg-image" alt="syncthing-new-folder-window" loading="lazy" width="1816" height="1070" srcset="/images/size/w600/2021/02/7.jpeg 600w, /images/size/w1000/2021/02/7.jpeg 1000w, /images/size/w1600/2021/02/7.jpeg 1600w, /images/2021/02/7.jpeg 1816w" sizes="(min-width: 720px) 720px"></figure><p>Give your new folder a name and change the path to be relative to the folder on your machine that you want to backup. I tend to keep all of my synced folders under <code>/home/ed/Syncthing</code> just so that I know where everything is.</p><figure class="kg-card kg-image-card"><img src="/images/2021/02/8.jpeg" class="kg-image" alt="syncthing-sharing-tab" loading="lazy" width="1413" height="452" srcset="/images/size/w600/2021/02/8.jpeg 600w, /images/size/w1000/2021/02/8.jpeg 1000w, /images/2021/02/8.jpeg 1413w" sizes="(min-width: 720px) 720px"></figure><p>Navigate to the <code>Sharing Tab</code></p><figure class="kg-card kg-image-card"><img src="/images/2021/02/9.jpeg" class="kg-image" alt="syncthing-share-with" loading="lazy" width="1843" height="657" srcset="/images/size/w600/2021/02/9.jpeg 600w, /images/size/w1000/2021/02/9.jpeg 1000w, /images/size/w1600/2021/02/9.jpeg 1600w, /images/2021/02/9.jpeg 1843w" sizes="(min-width: 720px) 720px"></figure><p>Here are the devices you have added. I have a couple more than you probably will just because this is my existing system. Simply select the device that you added and click <code>Save</code></p><h3 id="accepting-the-new-device">Accepting the new device</h3><figure class="kg-card kg-image-card"><img src="/images/2021/02/10.jpeg" class="kg-image" alt="syncthing-new-device" loading="lazy" width="2000" height="552" srcset="/images/size/w600/2021/02/10.jpeg 600w, /images/size/w1000/2021/02/10.jpeg 1000w, /images/size/w1600/2021/02/10.jpeg 1600w, /images/2021/02/10.jpeg 2000w" sizes="(min-width: 720px) 720px"></figure><p>Next, on <code>Syncthing B</code>, there will be a new call-to-action at the top of the page. This is a request for you to allow <code>Syncthing A</code> to add <code>Syncthing B</code> as a device. Simply click <code>Add</code>.</p><figure class="kg-card kg-image-card"><img src="/images/2021/02/11.jpeg" class="kg-image" alt="syncthing-add-folder" loading="lazy" width="1853" height="1286" srcset="/images/size/w600/2021/02/11.jpeg 600w, /images/size/w1000/2021/02/11.jpeg 1000w, /images/size/w1600/2021/02/11.jpeg 1600w, /images/2021/02/11.jpeg 1853w" sizes="(min-width: 720px) 720px"></figure><p>Then, shortly after accepting the new device, you'll get another popup which is <code>Syncthing A</code> requesting to share the folder you have just create.</p><p>Click <code>Add</code></p><figure class="kg-card kg-image-card"><img src="/images/2021/02/12.jpeg" class="kg-image" alt="syncthing-saved-folder" loading="lazy" width="1164" height="585" srcset="/images/size/w600/2021/02/12.jpeg 600w, /images/size/w1000/2021/02/12.jpeg 1000w, /images/2021/02/12.jpeg 1164w" sizes="(min-width: 720px) 720px"></figure><p>Finally, you now need to choose a location for this folder to sync to.</p><p>Click <code>Save</code> and that's it.</p><h3 id="testing">Testing</h3><p>Feel free to test by adding a file to the <code>Syncthing A</code> folder and wait for the <code>global state</code> to change on <code>Syncthing B</code></p><hr><h2 id="setting-up-other-syncthing-nodes">Setting up other Syncthing nodes</h2><p>Syncthing works on everything apart from iPhone. <a href="https://forum.syncthing.net/t/ios-iphone-crowdfunding-on-bountysource/13136">There are whispers of something coming to iOS in the future</a>, but not right now.</p><h3 id="setting-up-another-node-on-a-linux-machine">Setting up another node on a Linux machine</h3><p>Setting up a node on another Linux machine is as simple as just copying the steps above. If the machine isn't headless, such as a desktop machine, you can ignore the part about changing the ip address in the <code>config.xml</code></p><h3 id="setting-up-another-node-on-a-windows-machine">Setting up another node on a Windows machine</h3><p>Setting up on Windows is relativley easy. You can either go the manual route of <a href="https://syncthing.net/downloads/">downloading the Syncthing exe</a>. Or by opting for the simple solution which is installing <a href="https://github.com/canton7/SyncTrayzor">SyncTrayzor</a> which adds a system tray icon and runs Syncthing in the backgroud automatically.</p><h3 id="setting-up-another-node-on-an-android-device">Setting up another node on an Android device</h3><p>Syncthing has apps in the <a href="https://play.google.com/store/apps/details?id=com.nutomic.syncthingandroid">Play Store</a> and <a href="https://f-droid.org/packages/com.nutomic.syncthingandroid/">F-Droid</a> which can be downloaded and installed.</p><hr><h2 id="thanks">Thanks</h2><p>Let me know if this has helped in the comments below.</p><p>Also, feel free to check out my other blog posts <a href="https://theselfhostingblog.com/">here</a></p><p></p>
