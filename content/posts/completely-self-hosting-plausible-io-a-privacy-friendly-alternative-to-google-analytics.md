<figure class="kg-card kg-image-card kg-card-hascaption"><a href="https://bowlerdesign.co.uk?referrer=theselfhostingblog.com"><img src="/images/2021/09/Banner.png" class="kg-image" alt loading="lazy" width="500" height="350"></a><figcaption>Sponsored Post</figcaption></figure><h2 id="what-is-plausible">What is Plausible?</h2><p>Plausible is a lightweight, ethical, open-source and privacy-friendly Google Analytics alternative. </p><h3 id="google-analytics">Google Analytics</h3><p>Google Analytics collects more data than the average site owner needs to analyse their business, meaning that there is a lot of data being sent to the site owner that is unnecessary and will never be used. Plausible focuses only on the important web analytics data required for a business to get what it needs.</p><h3 id="tracking">Tracking</h3><p>The tracking data that Plausible collects are completely in our control. Your tracking details are not sold on to third-parties. Nor is it mined or harvested for personal or behavioural trends.</p><h3 id="open-source">Open-source</h3><p>Plausible is<a href="https://plausible.io/open-source-website-analytics"><strong> completely open-source</strong></a>, meaning that anybody can view their source code and evaluate what the company is doing behind the scenes. With Google Analytics, we have no way of knowing what's being processed on their servers. We don't need to trust Plausible, because they've made it so we don't have to.</p><h3 id="self-hosting">Self-hosting</h3><p>Plausible is also available for self-hosting! Which is amazing. Which we're going to be looking into in this post!</p><h2 id="self-hosting-plausible-io">Self-hosting Plausible.io</h2><h3 id="prerequisites-provisioning-your-server-">Prerequisites: Provisioning your server.</h3><p>Plausible is available to self-host by using Docker and Docker Compose. Plausible will run on any x86_64 architecture. Which unfortunately means we <em>currently </em>can't run Plausible on a <a href="https://amzn.to/3qvyjj2">Raspberry Pi</a>.</p><p>We can however run Plausible in <a href="https://www.digitalocean.com/">Digital Ocean</a>. This also happens to be recommended by the Plausible.io guys!</p><p>Using the following link will give you $100 worth of credits for 60 days to play around with, just sign up using <a href="https://m.do.co/c/d2a3afe52625"><strong><strong>this link</strong></strong></a>.</p><p>If you do end up using Digital Ocean, be sure to follow our tutorials on <a href="https://theselfhostingblog.com/posts/setting-up-ufw-on-ubuntu-server/"><strong>locking down your exposed ports with UFW</strong></a></p><h3 id="step-1-installing-docker-on-your-server">Step 1: Installing Docker on your server</h3><p>All of the following instructions are based on the Debian distro, so if you're running a server with Ubuntu, these instructions will be perfect for you. If not, you may have to adjust the commands below to suit your distro.</p><p>The first step is to just make sure our server is up to date. Run the following commands to pull down the latest updates from our distro repositories.</p><pre><code class="language-bash">sudo apt-get update &amp;&amp; sudo apt-get upgrade</code></pre><p>You should see an output like the following</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="/images/2021/02/image-7.png" class="kg-image" alt="Console output for running update and upgrade commands" loading="lazy"><figcaption>Console output for running the update and upgrade commands</figcaption></figure><p>Next, we need to install Docker. Docker is the layer which your containers run. Portainer is a UI that sits upon Docker to assist with container management.</p><p>To install Docker on your instance, you need to run the following command.</p><p>The following script is a convenience script <a href="https://docs.docker.com/engine/install/debian/#install-using-the-convenience-script"><strong>provided by the Docker team</strong></a>. It's highly recommended to always check what you're going to execute, before executing it.</p><figure class="kg-card kg-code-card"><pre><code class="language-bash">curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh</code></pre><figcaption>Installing Docker using the convenience script</figcaption></figure><p>Once you have executed the Docker install script. You should see an output like the following.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="/images/2021/02/image-8.png" class="kg-image" alt="Docker convenience script install output" loading="lazy"><figcaption>Docker convenience script install output</figcaption></figure><p>As you can see in the output, the command was executed successfully. You may also notice that there is a console message specifying how to use Docker as a non-root user.</p><p>This means that whenever you are executing the Docker command, you'll no longer need to type in your sudo password.</p><p>If this sounds good to you, you can simply run the provided command, substituting <code>your-user</code> for your server user. In my case, my user is <code>ubuntu</code>. My command would look like this.</p><figure class="kg-card kg-code-card"><pre><code class="language-bash">sudo usermod -aG docker ubuntu</code></pre><figcaption>Adding your user to the Docker group</figcaption></figure><p>We also need to install Docker Compose. This can be done by running the following commands.</p><pre><code class="language-bash">sudo curl -L "https://github.com/docker/compose/releases/download/1.28.5/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose</code></pre><h3 id="step-2-installing-plausible-io">Step 2: Installing Plausible.io</h3><p>It's now time to get Plausible up and running on our container 🎉</p><p>Let's start by making sure we're in the correct context on our server.</p><pre><code class="language-bash">cd</code></pre><p>This will make sure that we're in our Home folder for our user!</p><p>Next, let's clone Plausible's latest hosting repository. Run the following commands.</p><pre><code>git clone https://github.com/plausible/hosting
cd hosting</code></pre><p>You'll find 2 important files in this directory.</p><ul><li><code>docker-compose.yml</code> - This is the guts of Plausible. It comes with sensible defaults that are ready to go but feel free to change some options if you need to.</li><li><code>plausible-conf.env</code> - configures the Plausible server itself. Full configuration options are documented <a href="https://plausible.io/docs/self-hosting-configuration">here</a>.</li></ul><p>Part of the <code>plausible-conf.env</code> contains a line for <code>SECRET_KEY_BASE</code>. We'll need to generate a random base64 string for this value. This will be used to secure your app.</p><pre><code class="language-bash">openssl rand -base64 64</code></pre><p>You should get an output that looks something like the following.</p><pre><code>N5bBVFKyQI4JtYD4h5k/hd+sHi3h2uz/cO+RhCv0ed88+5an//dEW1kbnQCFOMhU
y0venoG8ZUdY/w0Ya2BStg==</code></pre><p>We now need to edit the <code>plausible-conf.env</code> file to configure our variables. Using your favourite editor.</p><pre><code class="language-bash">vim plausible-conf.env</code></pre><p>You will need to fill out the environment file with some values. Here's an example of my configuration.</p><p>Note: we'll need to specify an endpoint here that we want Plausible to run on, more on this later.</p><pre><code>ADMIN_USER_EMAIL=ed@theselfhostingblog.com
ADMIN_USER_NAME=Ed
ADMIN_USER_PWD=test
BASE_URL=https://plausible.theselfhostingblog.com
SECRET_KEY_BASE=N5bBVFKyQI4JtYD4h5k/hd+sHi3h2uz/cO+RhCv0ed88+5an//dEW1kbnQCFOMhU
y0venoG8ZUdY/w0Ya2BStg==</code></pre><h3 id="step-3-starting-plausible-io">Step 3: Starting Plausible.io</h3><p>That's all the config done for now! let's start it up.</p><pre><code class="language-bash">docker-compose up -d</code></pre><p>Eventually, you should see the following output.</p><pre><code class="language-bash">Creating hosting_mail_1                ... done
Creating hosting_plausible_db_1        ... done
Creating hosting_plausible_events_db_1 ... done
Creating hosting_plausible_1           ... done</code></pre><p>Now, we should be able to navigate to our service running on <code>localhost:8000</code> (or your IP address, if you're running in Digital Ocean)</p><p>Success!</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-4.png" class="kg-image" alt loading="lazy" width="1073" height="736" srcset="/images/size/w600/2021/03/image-4.png 600w, /images/size/w1000/2021/03/image-4.png 1000w, /images/2021/03/image-4.png 1073w" sizes="(min-width: 720px) 720px"></figure><h2 id="configuring-plausible-io">Configuring Plausible.io</h2><h3 id="logging-in">Logging in</h3><p>To log in, we just need to use the details that we created earlier in the <a href="#step-2-installing-plausible-io">Installing Plausible.io section</a>.</p><h3 id="activating-your-account">Activating your account</h3><p>Sweet, after logging in, we should have an email sent to us with the activation code.</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-5.png" class="kg-image" alt loading="lazy" width="1061" height="547" srcset="/images/size/w600/2021/03/image-5.png 600w, /images/size/w1000/2021/03/image-5.png 1000w, /images/2021/03/image-5.png 1061w" sizes="(min-width: 720px) 720px"></figure><p>Click on <code>Request activation code</code></p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-6.png" class="kg-image" alt loading="lazy" width="1124" height="662" srcset="/images/size/w600/2021/03/image-6.png 600w, /images/size/w1000/2021/03/image-6.png 1000w, /images/2021/03/image-6.png 1124w" sizes="(min-width: 720px) 720px"></figure><p>You should receive an email within a few seconds. Mine ended up in my spam folder. Enter the code that you receive and move on to the next section.</p><p>If you're having issues with receiving your activation email. You can use the command below to manually activate your account.</p><pre><code class="language-bash">docker exec hosting_plausible_db_1 psql -U postgres -d plausible_db -c "UPDATE users SET email_verified = true;"</code></pre><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-8.png" class="kg-image" alt loading="lazy" width="976" height="705" srcset="/images/size/w600/2021/03/image-8.png 600w, /images/2021/03/image-8.png 976w" sizes="(min-width: 720px) 720px"></figure><p> Enter your domain name for your site and the timezone that you're in.</p><p>Then move on to Adding the Snippet.</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-9.png" class="kg-image" alt loading="lazy" width="925" height="639" srcset="/images/size/w600/2021/03/image-9.png 600w, /images/2021/03/image-9.png 925w" sizes="(min-width: 720px) 720px"></figure><p>As you can see below, there's a javascript snippet that you need to add to the head of your website. This should be simple to do, whether it's adding it into your website HTML, or using code injection if your site is running on a CMS.</p><p>After adding your code, click on <code>Start collecting data</code>.</p><figure class="kg-card kg-image-card"><img src="/images/2021/03/image-10.png" class="kg-image" alt loading="lazy" width="1271" height="936" srcset="/images/size/w600/2021/03/image-10.png 600w, /images/size/w1000/2021/03/image-10.png 1000w, /images/2021/03/image-10.png 1271w" sizes="(min-width: 720px) 720px"></figure><p>Awesome! we're nearly there!</p><h2 id="configuring-nginx">Configuring Nginx</h2><p>We need to set up Nginx on our instance to allow us to point a domain at our Plausible.io instance.</p><h3 id="installing-nginx">Installing Nginx</h3><p>We first need to execute the following command to get Nginx installed.</p><pre><code class="language-bash">sudo apt-get install nginx</code></pre><p>If you have UFW installed, you will have to allow Nginx through your local firewall.</p><p>I have a tutorial for <a href="https://theselfhostingblog.com/posts/setting-up-ufw-on-ubuntu-server/"><strong>setting up UFW here</strong></a></p><figure class="kg-card kg-bookmark-card"><a class="kg-bookmark-container" href="https://theselfhostingblog.com/posts/setting-up-ufw-on-ubuntu-server/"><div class="kg-bookmark-content"><div class="kg-bookmark-title">Setting Up UFW on Ubuntu Server</div><div class="kg-bookmark-description">UFW is a program that allows you to internally control ports on your Linux instance. This gives you the ability to forward ports from your machine.</div><div class="kg-bookmark-metadata"><img class="kg-bookmark-icon" src="https://theselfhostingblog.com/favicon.png"><span class="kg-bookmark-author">The Self-Hosting Blog</span><span class="kg-bookmark-publisher">Ed Leeman</span></div></div><div class="kg-bookmark-thumbnail"><img src="https://images.unsplash.com/photo-1558494949-ef010cbdcc31?ixlib&#x3D;rb-1.2.1&amp;q&#x3D;80&amp;fm&#x3D;jpg&amp;crop&#x3D;entropy&amp;cs&#x3D;tinysrgb&amp;w&#x3D;2000&amp;fit&#x3D;max&amp;ixid&#x3D;eyJhcHBfaWQiOjExNzczfQ"></div></a></figure><p>To allow Nginx, run the following command.</p><pre><code class="language-bash">sudo ufw app list</code></pre><pre><code>Output
---

Available applications:
  Nginx Full
  Nginx HTTP
  Nginx HTTPS
  OpenSSH</code></pre><p>As you can see, there are three profiles available for Nginx:</p><ul><li><strong>Nginx Full</strong>: This profile opens both port 80 (normal, unencrypted web traffic) and port 443 (TLS/SSL encrypted traffic)</li><li><strong>Nginx HTTP</strong>: This profile opens only port 80 (normal, unencrypted web traffic)</li><li><strong>Nginx HTTPS</strong>: This profile opens only port 443 (TLS/SSL encrypted traffic)</li></ul><p>Run the following</p><pre><code class="language-bash">sudo ufw allow 'Nginx Full'</code></pre><p>The next thing to do is just double check your Nginx server is up and running</p><pre><code class="language-bash">sudo systemctl status nginx</code></pre><p>You should see something that looks like the following</p><pre><code>● nginx.service - A high performance web server and a reverse proxy server
   Loaded: loaded (/lib/systemd/system/nginx.service; enabled; vendor preset: enabled)
   Active: active (running)
...</code></pre><p>The next part allows us to take incoming traffic and point it to your container instance. Allowing you to point your domain at your Plausible.io instance.</p><p>Navigate to <code>/etc/nginx/</code></p><pre><code class="language-shell">cd /etc/nginx/
</code></pre><p>Use your favourite text editor and open the following file with sudo</p><pre><code class="language-shell">sudo vim nginx.conf
</code></pre><p>Paste in the template below. Being sure to substitute <code>example.com</code> with your domain that we configured back in the <a href="#step-2-installing-plausible-io">Installing Plausible.io section</a>.</p><pre><code class="language-shell">  
server {
	# replace example.com with your domain name
	server_name example.com;
	
	listen 80;
	listen [::]:80;

	location / {
		proxy_pass http://127.0.0.1:8000;
		proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
	}
}</code></pre><p>Save and quit, then restart Nginx.</p><pre><code class="language-bash">sudo systemctl restart nginx</code></pre><h3 id="port-forwarding">Port-forwarding</h3><p>You will need to port forward your instance to allow public access to your instance. This will involve googling how to port forward from your router.</p><p>You'll need to <strong><a href="https://portforward.com/">point port 80 and 443 to your instance</a></strong> where Nginx is set up.</p><h3 id="installing-certbot">Installing Certbot</h3><p>Certbot allows us to generate SSL certificates for free with Let's Encrypt. It's simple to install and use. Even hooks in with Nginx, meaning that there's no more manual configuration required.</p><p>To install Certbot, simply run the following command</p><pre><code class="language-bash">sudo apt-get install certbot python-certbot-nginx</code></pre><p>Then, to set up your SSL certificate, run</p><pre><code class="language-bash">sudo certbot</code></pre><p>Follow the instructions, select your domain name from the Nginx list. <br>Also, select <code>redirect</code> as this will upgrade any HTTP requests to HTTPS.</p><h2 id="wrapping-up">Wrapping up</h2><p>You should now be able to load up Plausible.io on your configured domain.
